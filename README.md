# 🧶 AmiPattern Web

Catálogo de patrones de amigurumi — versión web construida con React + Supabase.

---

## 📦 Stack

| Capa | Tecnología |
|------|------------|
| UI | React 18 |
| Base de datos | Supabase (PostgreSQL gratuito) |
| Almacenamiento de imágenes | Supabase Storage |
| Deploy | Vercel o Netlify (gratuito) |

---

## 🚀 Instalación local

```bash
# 1. Clona / copia el proyecto
cd amipattern-web

# 2. Instala dependencias
npm install

# 3. Copia el archivo de variables de entorno
cp .env.example .env.local

# 4. Edita .env.local con tus credenciales (ver siguiente sección)

# 5. Inicia el servidor de desarrollo
npm start
```

## 🗄️ Configurar Supabase (gratuito)

### Paso 1 — Crear cuenta y proyecto

1. Ve a [https://app.supabase.com](https://app.supabase.com)
2. Crea una cuenta gratuita (con GitHub es lo más rápido)
3. Clic en **New project**
4. Elige nombre, contraseña (guárdala) y región (elige la más cercana, ej. `us-east-1`)
5. Espera ~2 minutos mientras se crea el proyecto

### Paso 2 — Crear la tabla `patterns`

1. En tu proyecto, ve a **SQL Editor** → **New query**
2. Pega y ejecuta este SQL:

```sql
create table if not exists patterns (
  id          bigint generated by default as identity primary key,
  nombre      text        not null,
  emoji       text        not null default '🧶',
  dificultad  text        not null default 'Principiante',
  estado      text        not null default 'En progreso',
  talla       text,
  aguja       text,
  hilos       text[]      default '{}',
  materiales  text,
  notas       text,
  pasos       jsonb       default '[]',
  imagen_url  text,
  color       text        not null default '#FAD2E1',
  fecha       date        not null default now(),
  created_at  timestamptz default now(),
  updated_at  timestamptz default now()
);

-- Habilitar Row Level Security con acceso público
alter table patterns enable row level security;

create policy "Acceso público"
  on patterns for all
  using (true)
  with check (true);
```

### Paso 3 — Crear el bucket de imágenes

1. Ve a **Storage** → **New bucket**
2. Nombre: `pattern-images`
3. Activa **Public bucket** ✅
4. Clic en **Save**

Luego en **SQL Editor**, ejecuta:

```sql
create policy "Subida pública de imágenes"
  on storage.objects for insert
  with check (bucket_id = 'pattern-images');

create policy "Lectura pública de imágenes"
  on storage.objects for select
  using (bucket_id = 'pattern-images');

create policy "Borrar imágenes"
  on storage.objects for delete
  using (bucket_id = 'pattern-images');
```

### Paso 4 — Obtener las credenciales

1. Ve a **Settings** → **API**
2. Copia:
   - **Project URL** → es tu `REACT_APP_SUPABASE_URL`
   - **anon / public key** → es tu `REACT_APP_SUPABASE_ANON_KEY`
3. Pégalas en tu `.env.local`:

```env
REACT_APP_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxx.supabase.co
REACT_APP_SUPABASE_ANON_KEY=eyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
```

## ☁️ Deploy gratuito en Vercel

Vercel es la opción recomendada para proyectos React.

### Opción A — Deploy desde GitHub (recomendado)

1. Sube tu proyecto a GitHub:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/TU_USUARIO/amipattern-web.git
   git push -u origin main
   ```

2. Ve a [https://vercel.com](https://vercel.com) → **Add New Project**
3. Importa tu repositorio de GitHub
4. Vercel detecta automáticamente que es un proyecto React
5. Configuración de build:
   - **Build command:** `npm run build`
   - **Output Directory:** `build`
   - **Install Command:** `npm install`
5. **Antes de hacer clic en Deploy**, ve a **Environment Variables** y agrega:
   - `REACT_APP_SUPABASE_URL` → tu URL de Supabase
   - `REACT_APP_SUPABASE_ANON_KEY` → tu clave anon
   **En la seccion Build and Output Settings**
6. Clic en **Deploy** 🚀

**¡Listo!** Vercel te da una URL como `https://amipattern-web.vercel.app`

Cada vez que hagas `git push`, Vercel redeploya automáticamente.

### Opción B — Deploy con Vercel CLI

```bash
# Instalar Vercel CLI
npm install -g vercel

# Desde la carpeta del proyecto
vercel

# Seguir las instrucciones del CLI
# Cuando pregunte por variables de entorno, agrégalas ahí
```

## ☁️ Deploy gratuito en Netlify

### Opción A — Deploy desde GitHub

1. Sube tu proyecto a GitHub (igual que arriba)
2. Ve a [https://app.netlify.com](https://app.netlify.com) → **Add new site** → **Import from Git**
3. Conecta con GitHub y selecciona tu repo
4. Configuración de build:
   - **Build command:** `npm run build`
   - **Publish directory:** `build`
5. Clic en **Show advanced** → **New variable** y agrega:
   - `REACT_APP_SUPABASE_URL`
   - `REACT_APP_SUPABASE_ANON_KEY`
6. Clic en **Deploy site** 🚀

### Opción B — Drag & Drop (sin GitHub)

```bash
# Primero genera el build de producción
npm run build
```

1. Ve a [https://app.netlify.com/drop](https://app.netlify.com/drop)
2. Arrastra la carpeta `build/` al área de drop
3. ⚠️ Con esta opción las variables de entorno NO funcionan en el build.
   Para esto debes crear un archivo `build/.env` — **no recomendado para producción**.
   Usa la Opción A para poder agregar variables de entorno de forma segura.

## 🔐 Seguridad — Próximos pasos

Actualmente la app usa acceso público sin autenticación (cualquiera puede leer/escribir).
Para agregar autenticación con Supabase Auth:

1. Activa **Authentication** → **Email** en tu proyecto de Supabase
2. Cambia las políticas de RLS para que solo el dueño pueda editar sus patrones:

```sql
-- Elimina la política pública anterior
drop policy "Acceso público" on patterns;

-- Agrega columna de usuario
alter table patterns add column user_id uuid references auth.users default auth.uid();

-- Nuevas políticas por usuario
create policy "Ver patrones propios"
  on patterns for select using (auth.uid() = user_id);

create policy "Crear patrones"
  on patterns for insert with check (auth.uid() = user_id);

create policy "Editar patrones propios"
  on patterns for update using (auth.uid() = user_id);

create policy "Borrar patrones propios"
  on patterns for delete using (auth.uid() = user_id);
```
